Font-Weight Numeric Mapping Test (100-900)

Verifies CSS font-weight numeric values (100-900) map correctly to FreeType font selection.

1. Numeric Weight Values (100-900)

Thin 100 AaBbCc 123 Thin (100) - Extra light stroke
Extra Light 200 AaBbCc 123 Extra Light (200) - Ultra thin
Light 300 AaBbCc 123 Light (300) - Thin stroke
Regular 400 AaBbCc 123 Regular (400) - Normal
Medium 500 AaBbCc 123 Medium (500) - Medium-bold
Semi Bold 600 AaBbCc 123 Semi Bold (600) - Bold-ish
Bold 700 AaBbCc 123 Bold (700) - Bold stroke
Extra Bold 800 AaBbCc 123 Extra Bold (800) - Extra heavy
Black 900 AaBbCc 123 Black (900) - Heavyest stroke

2. Weight Value Normalization (Non-100 multiples)

Values are normalized to nearest 100 (e.g., 150→200, 349→300)

Input:150 150 AaBbCc 123 150 normalizes to 200 (Extra Light)
Input:349 349 AaBbCc 123 349 normalizes to 300 (Light)
Input:450 450 AaBbCc 123 450 normalizes to 500 (Medium)
Input:750 750 AaBbCc 123 750 normalizes to 800 (Extra Bold)

3. Keyword Values and Their Numeric Equivalents

normal →400 AaBbCc 123 "normal" maps to 400 (Regular)
bold →700 AaBbCc 123 "bold" maps to 700 (Bold)

4. Edge Cases (Clamping)

Values are clamped to 100-900 range

Input:0 0 AaBbCc 123 0 clamps to 100 (Thin)
Input:50 50 AaBbCc 123 50 clamps to 100 (Thin)
Input:1000 1000 AaBbCc 123 1000 clamps to 900 (Black)
