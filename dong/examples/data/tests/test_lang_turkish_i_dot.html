Turkish i/İ Language Test

This test demonstrates how the lang attribute affects Turkish character shaping, particularly the distinction between dotted i (i) and dotless i (ı), and dotted I (İ) and dotless I (I).

Turkish: lang="tr"
Turkish (tr)
i İ
dotted i, dotted I
ı I
dotless i, dotless I
English (en)
i I
dotted i, dotless I
ı İ
dotless i, dotted I
Turkish text example: "İstanbul'da ılık bir iklim var."
In Turkish language context (lang="tr"), the text shaping engine should:
  • Use dotted i (i) and dotted I (İ) as separate characters
  • Use dotless i (ı) and dotless I (I) as separate characters
  • Maintain proper case conversion: i → İ, I → ı
  • Apply Turkish-specific typographic rules
English: lang="en"
English text example: "Istanbul has a mild climate."
In English language context (lang="en"), the text shaping engine should:
  • Use standard Latin character set
  • Treat i/I as case pairs without Turkish-specific rules
  • Apply English typographic conventions
Mixed language content
English: "I like Istanbul."
Turkish: "İstanbul'u seviyorum."
Back to English: "The city is beautiful."
When mixing languages, each span with its own lang attribute should receive appropriate text shaping:
  • English spans use standard Latin shaping
  • Turkish spans use Turkish-specific shaping
  • Language inheritance works correctly for nested elements
Turkish case sensitivity test
Lowercase: i → İ (dotted I)
Uppercase: I → ı (dotless i)
This demonstrates the Turkish-specific case conversion rules that should be applied when lang="tr" is set:
  • Lowercase i converts to uppercase İ (dotted)
  • Uppercase I converts to lowercase ı (dotless)
  • This differs from English where i/I are simple case pairs
