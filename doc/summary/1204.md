# 项目架构与问题追查总结 (2025-12-04)

## 一、项目关键架构

### 1.1 技术栈

| 模块 | 技术 |
|------|------|
| 构建系统 | Zig Build |
| 图形后端 | SDL GPU |
| 字体加载 | FreeType |
| 文本塑形 | HarfBuzz |
| 字形渲染 | msdfgen (MSDF) |
| 布局引擎 | Yoga (Flex) + 自定义 Block/Inline |

### 1.2 核心模块

```
dong/src/render/
├── glyph_atlas.cpp      # MSDF 字形图集生成
├── gpu_driver_sdl.cpp   # GPU 渲染驱动，字形绘制
├── painter.cpp          # 绘制命令生成，baseline 计算
├── text_shaper.cpp      # HarfBuzz 文本塑形
└── font_resolver.cpp    # 字体解析与选择
```

### 1.3 渲染流程

```
CSS 样式 → StyleEngine → LayoutEngine (Yoga)
                              ↓
文本内容 → TextShaper (HarfBuzz) → GlyphAtlas (msdfgen)
                              ↓
                         Painter → DisplayList → GPUDriver (SDL)
```

## 二、当前追查问题

### 2.1 问题现象

文字纵向布局错误：
- 以 "dog" 为例，字形位置有纵向偏移
- 修复 y 轴翻转后，字形变成上下颠倒

### 2.2 问题分析

**坐标系差异**：
- msdfgen：y 轴向上（y=0 在底部）
- GPU 纹理：y 轴向下（y=0 在顶部）
- 屏幕坐标：y 轴向下

**关键变量**：
- `bearing_y_units`：基线到字形顶部的距离（来自 `bounds.t`）
- `padding_units`：MSDF padding 在 design units 中的大小
- `glyph_y = pen_y_px - bearing_y_px`：字形在屏幕上的 y 坐标

### 2.3 已尝试的修复

**文件**：`dong/src/render/glyph_atlas.cpp` (约第 585 行)

**修改**：
```cpp
// 修复前
const float* pixel = msdf(x, y);

// 修复后（导致字形颠倒）
const float* pixel = msdf(x, msdf_size - 1 - y);
```

**结果**：字形上下颠倒，说明需要同时调整其他相关计算。

### 2.4 待解决

需要找到正确的坐标系转换方案：

1. **方案 A**：保持 bitmap 不翻转，调整 UV 坐标
2. **方案 B**：翻转 bitmap，同时调整 `bearing_y` 计算
3. **方案 C**：在渲染时翻转 UV 的 v0/v1

关键是确保以下三者一致：
- MSDF 纹理内容方向
- `bearing_y` 的语义
- 渲染时的 `glyph_y` 计算
