# 字体渲染问题调试方法论

## 1. 问题定位思路

### 1.1 分层排查
字体渲染涉及多个层次，按数据流顺序排查：

1. **字体加载层** - FreeType/HarfBuzz 是否正确解析字体文件
2. **字形生成层** - MSDF 生成是否正确（形状、尺寸、距离场）
3. **纹理管理层** - Atlas 打包、UV 坐标计算是否正确
4. **渲染层** - Shader 采样、顶点坐标是否正确

### 1.2 隔离变量
- 对比正常字符和异常字符的差异
- 使用简单测试用例（如单个字符）排除干扰
- 检查同类字符（如两个 "l"）表现是否一致

## 2. 数据流追踪

### 2.1 关键数据点
在字体渲染中，需要追踪以下关键数据：

```
字体度量 (Design Units)
    ↓ scale 转换
像素尺寸 (Pixels)
    ↓ MSDF 生成
纹理尺寸 (Texture Size)
    ↓ UV 计算
UV 坐标 (0.0 ~ 1.0)
    ↓ 顶点生成
渲染矩形 (Screen Pixels)
```

### 2.2 一致性检查
**核心原则：UV 坐标覆盖的纹理区域必须与渲染矩形成比例对应**

- 如果 UV 覆盖整个纹理 (0,0)→(1,1)，渲染矩形应对应整个纹理尺寸
- 如果字形只占纹理的一部分，要么裁剪 UV，要么裁剪纹理输出

## 3. 调试技巧

### 3.1 添加诊断日志
在关键节点输出数据：

```cpp
LOG_DEBUG("glyph '{}': design_units({}, {}), pixels({}, {}), texture({}, {})",
          character, width_du, height_du, width_px, height_px, tex_w, tex_h);
```

### 3.2 可视化中间结果
- 导出 MSDF 纹理为图片，检查距离场是否正确
- 绘制字形边界框，验证位置和尺寸
- 使用不同颜色标记不同来源的数据

### 3.3 对比分析
找到一个正常工作的字符和异常字符，对比所有参数：

| 参数 | 正常字符 'H' | 异常字符 'l' |
|------|-------------|-------------|
| 设计单位宽高 | 1366 x 1462 | 569 x 1462 |
| MSDF scale | 0.0328 | 0.0328 |
| 纹理尺寸 | 64 x 64 | 64 x 64 |
| 实际占用 | ~53 x 56 | ~27 x 56 |

## 4. 常见问题模式

### 4.1 尺寸不匹配
**症状**：字符变形、拉伸、压缩、模糊
**原因**：UV 坐标与渲染矩形比例不一致
**检查**：`texture_aspect_ratio == render_rect_aspect_ratio`

### 4.2 内存未初始化
**症状**：随机噪点、条纹
**原因**：Bitmap 分配后未清零
**修复**：显式初始化 `memset` 或使用初始化构造

### 4.3 坐标系混淆
**症状**：字符上下颠倒、位置偏移
**原因**：Y 轴方向不一致（屏幕坐标 vs 纹理坐标）
**检查**：确认各层的坐标系约定

### 4.4 边界处理
**症状**：字符边缘被截断、出现黑边
**原因**：Padding 计算错误或纹理边界采样问题
**检查**：确保 padding 在各层一致传递

## 5. 验证方法

### 5.1 单元测试思维
- 测试极端情况：非常窄的字符（l, i）、非常宽的字符（W, M）
- 测试边界值：刚好填满纹理、远小于纹理

### 5.2 回归测试
- 修复后确保原本正常的字符仍然正常
- 测试不同字体、不同字号

### 5.3 数学验证
手算关键公式，确认代码实现正确：

```
scale = (msdf_size - 2 * range) / max(width, height)
actual_width = width * scale + 2 * range
uv_max = actual_width / texture_width  // 如果未裁剪
```

## 6. 资料查阅

### 6.1 官方文档优先
- msdfgen: https://github.com/Chlumsky/msdfgen
- FreeType: https://freetype.org/freetype2/docs/
- HarfBuzz: https://harfbuzz.github.io/

### 6.2 理解原理
MSDF 核心原理：
- 每个像素存储到最近边缘的有符号距离
- 三通道分别存储到不同边缘的距离
- Shader 中通过 `median(r, g, b)` 重建形状

### 6.3 参考实现
查看成熟项目的实现方式：
- Slug 库的文档
- ImGui 的字体渲染
- 游戏引擎（Unity/Unreal）的 SDF 字体实现

## 7. 总结

1. **系统性思考** - 理解完整数据流，不要只看局部
2. **数据驱动** - 用实际数值说话，不要猜测
3. **对比分析** - 找到正常和异常的差异点
4. **一致性原则** - 各层数据必须保持比例一致
5. **验证闭环** - 修复后必须验证，包括回归测试
