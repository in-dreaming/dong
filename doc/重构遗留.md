
重要：
- 核心思路：梳理重构整个架构，不留技术债。
- 原则：公共模块留在 core，通过能力注入隔离平台差异；backend 只做平台绑定与执行链。
- 禁止：为了“去 SDL”而把公共模块整体下沉到 backend（会导致每 backend 复制一份）。

## 二、方案1：边界与能力注入（摘要）

> 详见新增文档：`doc/架构重构方案1_总体框架.md`

### 2.1 核心边界
- core：DOM/CSS/Layout/Script + DisplayList + GPU IR + ResourceManager + **GlyphAtlas（公共）**
- backend：窗口/输入/GPUDevice/Shader 管理 + GPUCommandList 执行 + 平台能力

### 2.2 必需的注入能力（平台无关）
- 已有：`DongFileSystem`、`DongLogger`、`DongImageDecoder`、`DongGPUDriver`、`DongSurfaceFactory`
- 方案1新增/强化：
  - **GPUTexture/GPUResource 能力语义补齐**：upload 支持 stride/subrect + 可选 fence/ticket
  - **AtlasFactory 能力**：core 通过工厂创建 `DongImageAtlas`，避免直接调用 `dong_sdl_image_atlas_create`

### 2.3 TODO：GPU 压缩（片元/compute）
- 当前：RGBA8 → `DongImageDecoder.encode()`（CPU）→ `DongImageAtlas.upload()`
- TODO：RGBA8 → GPU shader/compute → 直接写入压缩 atlas/texture（避免 CPU blocks 中转）
