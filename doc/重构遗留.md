## 文件拆分方案

### 1. `layout_engine.cpp` (3526行) → 拆分为5个文件

```
src/layout/
├── layout_engine.cpp          # 主入口, Engine类核心 (~500行)
├── layout_block.cpp           # 块级格式化上下文 (~400行)
├── layout_inline.cpp          # 行内格式化上下文 (~600行)
├── layout_positioned.cpp      # 定位元素布局 (~400行)
├── layout_yoga_bridge.cpp     # Yoga样式映射 (~500行)
└── layout_intrinsic.cpp       # 固有尺寸计算 (~400行)
```

### 2. `css_parser.cpp` (2268行) → 拆分为4个文件

```
src/dom/css/
├── css_parser.cpp             # 主解析器 (~600行)
├── css_property_handlers.cpp  # 属性处理器注册 (~500行)
├── css_value_parser.cpp       # 值解析 (颜色/长度/etc) (~600行)
└── css_gradient_parser.cpp    # 渐变/滤镜/变换解析 (~400行)
```

### 3. `sdl_gpu_driver_execute.cpp` (1996行) → 拆分为4个文件

```
backends/sdl/
├── sdl_gpu_driver_execute.cpp      # 主执行循环 (~400行)
├── sdl_gpu_driver_draw_shapes.cpp  # 形状绘制命令 (~400行)
├── sdl_gpu_driver_draw_text.cpp    # 文本绘制命令 (~400行)
└── sdl_gpu_driver_layers.cpp       # 层管理命令 (~500行)

### 4. `js_bindings.cpp` (2240行) → 拆分为4个文件

```
src/script/
├── js_bindings.cpp            # 主绑定类 (~400行)
├── js_dom_bindings.cpp        # DOM节点/元素绑定 (~600行)

## 重构原则

1. **保持接口稳定**: 拆分时不改变公共API
2. **增量进行**: 每次只拆一个文件，确保编译通过再继续
3. **测试覆盖**: 拆分前后运行 `zig build run-feature-tests`
4. **命名一致**: 新文件名反映其职责
