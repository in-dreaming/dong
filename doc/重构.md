# Dong 引擎重构方案

## 目标

参考 Ultralight 架构，对 Dong 引擎进行重构，实现：

1. **平台隔离**：核心引擎 (dong.dll) 完全与平台无关，不依赖 SDL
2. **自定义 GPU Driver**：支持用户提供自定义 GPU 渲染后端
3. **自定义 Surface**：支持 CPU 渲染模式下的自定义像素缓冲区
4. **简化开发体验**：提供公共设施减少 demo 开发复杂度

---

## 一、当前架构分析

### 1.1 现有架构

```
dong.dll (核心)
├── DOM/CSS/Layout 引擎 (平台无关 ✓)
├── JavaScript 引擎 QuickJS (平台无关 ✓)
├── 渲染器 (SDL_GPU 依赖 ✗)
└── 字体/图像加载 (部分 SDL 依赖 ✗)

dong_plugin_sdl.dll (平台插件)
├── 窗口管理
├── 输入处理
├── GPU 渲染 (SDL_GPU)
└── 视频解码 (FFmpeg)

dong_legacy.lib (过渡库)
└── SDL-coupled GPU 渲染
```

### 1.2 当前问题

1. **渲染器与 SDL 耦合**
   - `dong_view_render_to_gpu_texture()` 直接返回 `SDL_GPUTexture*`
   - 无法使用 DirectX/Vulkan/Metal 原生 API

2. **Demo 复杂度过高**
   - `3d_screen_script.cpp` 需要 1500+ 行代码处理：
     - SDL 初始化和窗口创建
     - 着色器编译 (HLSL → SPIRV/DXIL)
     - GPU 管线创建
     - 顶点缓冲区管理
     - 深度纹理创建
     - HUD 系统实现
     - 射线检测和交互
   - 这些应由引擎/工具库提供

3. **缺少统一抽象**
   - 没有 Platform/Renderer/Surface 抽象接口
   - 无法注入自定义实现

### 2.2 核心组件

| 组件 | 职责 | 依赖 |
|------|------|------|
| **dong.dll** | DOM/CSS/Layout/Script + 渲染命令生成 | 无平台依赖 |
| **Platform** | 全局单例，注入 GPUDriver/Surface 等 | 纯接口 |
| **GPUDriver** | GPU 渲染后端抽象 | 用户实现 |
| **Surface** | CPU 渲染目标抽象 | 用户实现 |
| **SurfaceFactory** | Surface 生命周期管理 | 用户实现 |
| **FileSystem** | 文件 I/O 抽象 | 用户实现 |
| **AppCore** | 便捷封装 (窗口/输入/默认渲染) | SDL 或其他 |

### 3.2 GPUDriver 接口

```c
// dong_gpu_driver.h

typedef struct dong_gpu_driver_t dong_gpu_driver_t;

// 纹理句柄（由 Driver 实现定义）
typedef uint32_t dong_texture_id_t;

// 渲染缓冲区句柄
typedef uint32_t dong_render_buffer_id_t;

// 命令类型
typedef enum dong_gpu_command_type_t {
    DONG_GPU_CMD_CREATE_TEXTURE,
    DONG_GPU_CMD_UPDATE_TEXTURE,
    DONG_GPU_CMD_DESTROY_TEXTURE,
    DONG_GPU_CMD_CREATE_RENDER_BUFFER,
    DONG_GPU_CMD_DESTROY_RENDER_BUFFER,
    DONG_GPU_CMD_SET_RENDER_TARGET,
    DONG_GPU_CMD_CLEAR,
    DONG_GPU_CMD_DRAW_GEOMETRY,
} dong_gpu_command_type_t;

// GPU 命令结构
typedef struct dong_gpu_command_t {
    dong_gpu_command_type_t type;
    union {
        struct { /* CreateTexture params */ } create_texture;
        struct { /* UpdateTexture params */ } update_texture;
        struct { /* DrawGeometry params */ } draw_geometry;
        // ... 其他命令
    };
} dong_gpu_command_t;

// 命令列表
typedef struct dong_gpu_command_list_t {
    dong_gpu_command_t* commands;
    uint32_t count;
} dong_gpu_command_list_t;

// GPUDriver 虚表
typedef struct dong_gpu_driver_vtable_t {
    // 状态管理
    void (*begin_sync)(void* user);  // 开始同步（可选）
    void (*end_sync)(void* user);    // 结束同步（可选）

    // 纹理操作
    dong_texture_id_t (*create_texture)(void* user, uint32_t width, uint32_t height);
    void (*update_texture)(void* user, dong_texture_id_t texture,
                          const uint8_t* pixels, uint32_t width, uint32_t height);
    void (*destroy_texture)(void* user, dong_texture_id_t texture);
    void (*bind_texture)(void* user, uint8_t slot, dong_texture_id_t texture);

    // 渲染缓冲区（用于离屏渲染）
    dong_render_buffer_id_t (*create_render_buffer)(void* user,
                                                    uint32_t width, uint32_t height);
    void (*destroy_render_buffer)(void* user, dong_render_buffer_id_t buffer);
    void (*set_render_buffer_viewport)(void* user, uint32_t width, uint32_t height);

    // 几何体操作
    void (*create_geometry)(void* user, uint32_t geometry_id,
                           const float* vertices, uint32_t vertex_count,
                           const uint32_t* indices, uint32_t index_count);
    void (*update_geometry)(void* user, uint32_t geometry_id,
                           const float* vertices, uint32_t vertex_count);
    void (*destroy_geometry)(void* user, uint32_t geometry_id);

    // 命令执行
    void (*update_command_list)(void* user, const dong_gpu_command_list_t* list);
} dong_gpu_driver_vtable_t;

struct dong_gpu_driver_t {
    dong_gpu_driver_vtable_t vtable;
    void* user_data;
};
```

### 3.3 Surface 接口

```c
// dong_surface.h

typedef struct dong_surface_t dong_surface_t;
typedef struct dong_surface_factory_t dong_surface_factory_t;

// Surface 接口（CPU 渲染目标）
typedef struct dong_surface_vtable_t {
    uint32_t (*width)(void* user);
    uint32_t (*height)(void* user);
    uint32_t (*row_bytes)(void* user);  // 每行字节数
    size_t   (*size)(void* user);       // 总字节数

    // 像素访问（锁定/解锁模式，支持 GPU 同步）
    void*    (*lock_pixels)(void* user);
    void     (*unlock_pixels)(void* user);

    // 大小变化
    void     (*resize)(void* user, uint32_t width, uint32_t height);

    // 脏区域跟踪
    void     (*set_dirty_bounds)(void* user, int x, int y, int w, int h);
    void     (*get_dirty_bounds)(void* user, int* x, int* y, int* w, int* h);
    void     (*clear_dirty_bounds)(void* user);
} dong_surface_vtable_t;

struct dong_surface_t {
    dong_surface_vtable_t vtable;
    void* user_data;
};

// Surface 工厂接口
typedef struct dong_surface_factory_vtable_t {
    dong_surface_t* (*create_surface)(void* user, uint32_t width, uint32_t height);
    void (*destroy_surface)(void* user, dong_surface_t* surface);
} dong_surface_factory_vtable_t;

struct dong_surface_factory_t {
    dong_surface_factory_vtable_t vtable;
    void* user_data;
};
```

### 3.4 FileSystem 接口

## 四、Renderer 与 View 架构

### 4.1 渲染模式

支持两种渲染模式（参考 Ultralight）：

| 模式 | 描述 | 输出 | 适用场景 |
|------|------|------|----------|
| **CPU** | 软件光栅化 | Surface (像素缓冲区) | 简单 UI、低 GPU 负载 |
| **GPU** | 硬件加速 | RenderBuffer (离屏纹理) | 复杂 UI、动画 |

### 4.2 Renderer

```c

typedef struct dong_view_t dong_view_t;

    double device_scale;     // 设备像素比
```

## 六、游戏引擎集成模式

### 6.1 典型集成流程

```cpp
// 1. 初始化 Platform（注入自定义 Driver）
dong_platform_t* platform = dong_platform_instance();

MyGPUDriver* gpu_driver = new MyGPUDriver(my_d3d12_device);
dong_platform_set_gpu_driver(platform, gpu_driver->to_c_interface());

MyFileSystem* fs = new MyFileSystem(my_asset_manager);
dong_platform_set_file_system(platform, fs->to_c_interface());

// 2. 创建 Renderer
dong_renderer_config_t cfg = {};
cfg.is_accelerated = 1;
dong_renderer_t* renderer = dong_renderer_create(&cfg);

// 3. 创建 View
dong_view_config_t view_cfg = {};
view_cfg.width = 800;
view_cfg.height = 600;
view_cfg.is_accelerated = 1;
dong_view_t* view = dong_view_create(renderer, &view_cfg);

// 4. 加载内容
dong_view_load_html(view, "<html>...</html>");

// 5. 游戏主循环中
void GameLoop() {
    // 更新（尽可能频繁调用）
    dong_renderer_update(renderer);

    // 每帧渲染前
    dong_renderer_refresh_display(renderer, 0);
    dong_renderer_render(renderer);

    // 获取纹理（通过自定义 GPUDriver 管理）
    dong_render_buffer_id_t texture_id = dong_view_get_render_target(view);

    // 在你的渲染管线中使用该纹理
    DrawUIQuad(gpu_driver->get_native_texture(texture_id));
}

// 6. 输入处理
void HandleInput(InputEvent& e) {
    dong_mouse_event_t mouse = {};
    mouse.type = DONG_MOUSE_EVENT_MOUSE_MOVE;
    mouse.x = e.x;
    mouse.y = e.y;
    dong_view_fire_mouse_event(view, &mouse);
}
```

### 6.2 简化后的 Demo

对比重构前后的 3d_screen_script 代码量：

**重构前** (~1500 行):
- SDL 初始化
- 着色器编译
- 管线创建
- 深度纹理创建
- 顶点缓冲区管理
- HUD 系统
- 输入处理
- 射线检测
- 相机控制

**重构后** (~200 行):
```cpp
int main() {
    // 使用 AppCore 简化版
    dong_app_settings_t settings = {};
    settings.title = "Dong 3D Demo";
    settings.width = 1920;
    settings.height = 1080;

    dong_app_t* app = dong_app_create(&settings);
    dong_renderer_t* renderer = dong_app_get_renderer(app);

    // 创建 3D 场景管理器（新增工具类）
    Dong3DScene* scene = dong_3d_scene_create(app);

    // 添加 HTML 屏幕（工具类处理变换/射线检测/渲染）
    DongScreen3D* screen1 = dong_3d_scene_add_screen(scene,
        "screen1.html", 800, 600, Vec3{0, 2, -3});
    DongScreen3D* screen2 = dong_3d_scene_add_screen(scene,
        "screen2.html", 960, 540, Vec3{4, 2, -3});

    // 添加 HUD
    dong_overlay_t* hud = dong_overlay_create(app, 400, 100);
    dong_view_load_html(dong_overlay_get_view(hud), "<div id='fps'>FPS: 0</div>");

    // 主循环
    while (dong_app_is_running(app)) {
        dong_3d_scene_update(scene, delta_time);
        dong_3d_scene_handle_input(scene);  // 自动处理射线检测和交互
        dong_3d_scene_render(scene);
    }

    dong_3d_scene_destroy(scene);
    dong_overlay_destroy(hud);
    dong_app_destroy(app);
    return 0;
}
```

## 附录 A: Ultralight 关键设计参考

### A.1 GPUDriver 命令模式

Ultralight 使用命令列表模式，引擎生成命令，Driver 执行：

```cpp
// 引擎侧
void Renderer::Render() {
    for (auto& view : views_) {
        if (view->needs_paint()) {
            BuildCommandList(view);
        }
    }
    driver_->UpdateCommandList(command_list_);
}

// Driver 侧
void MyGPUDriver::UpdateCommandList(const CommandList& list) {
    for (auto& cmd : list.commands) {
        switch (cmd.type) {
            case kCommandType_DrawGeometry:
                ExecuteDraw(cmd.geometry_id, cmd.indices_count);
                break;
            // ...
        }
    }
}
```

### A.2 Surface 锁定模式

```cpp
// CPU 渲染后，用户上传到 GPU
if (!surface->dirty_bounds().IsEmpty()) {
    void* pixels = surface->LockPixels();
    MyTexture->Upload(pixels, surface->row_bytes() * surface->height());
    surface->UnlockPixels();
    surface->ClearDirtyBounds();
}
```

### A.3 脏区域优化

只重绘变化区域，大幅减少 GPU 带宽：

```cpp
IntRect dirty = surface->dirty_bounds();
if (!dirty.IsEmpty()) {
    // 只更新脏区域
    UpdateTextureRegion(dirty.x, dirty.y, dirty.width, dirty.height);
}
```

## 结论

本重构方案参考 Ultralight 的成熟架构，通过 Platform 单例 + 依赖注入模式，实现：

1. **核心引擎平台无关**：dong.dll 不依赖 SDL
2. **自定义渲染后端**：GPUDriver 接口支持任意 GPU API
3. **CPU/GPU 双模式**：Surface 接口支持软件渲染
4. **简化开发体验**：AppCore + 工具类大幅减少代码量
