

Profiling 完成，优化已实施。总结：

## Profiling 结果与优化

**根因**：100% 的帧时间花在 GPU 执行路径上。即使场景完全静态（没有任何 dirty 状态），每帧仍然重新提交所有 GPU draw call（~50 个 scene node 的 rect/text 绘制命令），耗时约 6.5ms/帧。

**优化方案**：利用 GPU driver 已有的 **present-only 快速路径**。关键改动在 `engine_view.cpp`：

1. 新增 `commands_regenerated_this_tick_` 标志，跟踪当前帧是否真正重新生成了 GPU 命令
2. 在 `tickGenerateCommandsIfNeeded()` 中设置该标志
3. 在 `tickExecuteGPUCommandsIfReady()` 中，若无命令变化，传递空 command list 触发 **intermediate → swapchain blit** 快速路径，跳过所有 draw call

**效果**：

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| GPU 耗时/帧 | ~6500 us | ~420 us | **15x** |
| 总帧耗时 | ~6800 us | ~430 us | **16x** |
| 理论 FPS | ~150 | ~2300+ | **15x** |

这个优化对所有静态/低变化场景通用，不限于 Scene Graph 模式。当 UI 有交互变化时，会自动回到完整渲染路径。你可以运行 `dong_app.exe --html data\gamelikeui\game_ui2.html` 查看实际 FPS。
