# 分析 TypeScript 模块导入情况

深入分析指定路径下的 TypeScript/JavaScript 模块导入和依赖关系。

## 参数说明

用户可能会提供一个路径参数，例如：
- `/analyze-imports packages/apps/auto-ui/src`
- `/analyze-imports src/components`

如果没有提供路径，默认分析 `packages/apps/auto-ui/src` 目录。

## 执行策略

**优先使用 TypeScript 分析脚本**（更精确，功能更强）：

1. 首先尝试运行 `.claude/scripts/analyze-imports.ts` 脚本：
   ```bash
   tsx .claude/scripts/analyze-imports.ts [目标路径]
   ```

2. 如果脚本运行成功，直接输出结果给用户

3. 如果脚本不存在或运行失败（如缺少依赖），则回退到手动分析模式

## 手动分析模式

如果脚本不可用，使用 Claude 的工具手动分析：

### 1. 扫描文件
使用 Glob 工具查找所有 TypeScript/JavaScript 文件：
- 模式: `**/*.{ts,tsx,js,jsx}`
- 排除: `**/node_modules/**`, `**/dist/**`, `**/build/**`, `**/*.test.*`, `**/*.spec.*`

### 2. 提取导入语句
使用 Grep 工具提取导入：
- 正则: `^import\s+.*from\s+['"].*['"]` （基础导入）
- 正则: `^export\s+.*from\s+['"].*['"]` （重导出）
- 选项: `-n` 显示行号, `output_mode: "content"`

### 3. 分类统计
将导入分为以下类别：
- **外部依赖**: 不以 `.` 或 `@/` 开头的包（来自 node_modules）
- **内部相对路径**: 以 `./` 或 `../` 开头
- **路径别名**: 以 `@/` 等别名开头
- **类型导入**: 包含 `import type` 的语句

### 4. 分析维度
统计以下指标：
- 文件总数
- 导入语句总数
- 各外部包的引用频率（Top 15）
- 深层相对路径（3层及以上 `../`）的数量和位置
- 类型导入的比例

### 5. 生成报告
使用以下格式输出分析结果：

```
📊 导入统计概览
==================
📁 分析的文件总数: XXX
📦 导入语句总数: XXX
🌐 外部依赖数量: XXX
🔗 内部模块数量: XXX

📦 外部依赖排名（Top 15）
==================
| 排名 | 包名 | 引用次数 |
|------|------|----------|
| 1    | react | 120 |
...

⚠️ 潜在问题
==================
❗ 深层相对路径导入（3层及以上）: X 处
  示例:
  - src/foo/bar/baz.ts → import from '../../../utils/helper'

💡 优化建议
==================
1. 建议使用路径别名替代深层相对导入
2. ...
```

## 输出要求

- 使用清晰的中文表述
- 数据使用表格或列表呈现
- 突出显示关键发现和问题
- 提供具体可执行的优化建议
- 如果发现严重架构问题，优先指出

## 注意事项

- 对于大型项目（>500个文件），先询问是否继续完整分析
- 使用 Grep 和 Glob 工具并行扫描以提高效率
- 避免读取过多的文件内容，优先使用模式匹配
- 如果用户需要更详细的某个模块分析，可以进行二次深入分析
