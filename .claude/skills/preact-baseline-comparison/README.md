# Preact/React Baseline Comparison Skill

## Quick Start

This skill is automatically registered with Claude Code. To use it, reference it in your prompts:

```
/preact-baseline-comparison

Analyze the discovery-buttons baseline comparison results from the latest run.
HTML file: zig-out/bin/data/preact-discovery-buttons/index.html
Report JSON: zig-out/tmp/preact_baseline/discovery-buttons/discovery-buttons_report.json
```

## Tool: run_preact_baseline_compare.py

The companion CLI tool orchestrates baseline generation, Dong rendering, and LLM-powered diff analysis for all Preact/React components.

### Installation & Setup

No special installation needed—uses existing tools:
- `html_baseline_render.py` (Playwright + Chrome)
- `vl_tool_multi.py` (multi-frame visual diff with LLM)
- `html_render_test` (Dong rendering engine)

### Usage Examples

#### Run single test with visualization
```bash
cd dong/
python scripts/tools/run_preact_baseline_compare.py --case discovery-buttons --llm
```

Output files:
- `zig-out/tmp/preact_baseline/discovery-buttons/discovery-buttons_base.png` - Chrome baseline
- `zig-out/tmp/preact_baseline/discovery-buttons/discovery-buttons_f0.bmp` - Dong frame 0
- `zig-out/tmp/preact_baseline/discovery-buttons/discovery-buttons_f1.bmp` - Dong frame 1
- ... (up to f4 by default, configurable with `--frames`)
- `zig-out/tmp/preact_baseline/discovery-buttons/discovery-buttons_merged.png` - Side-by-side comparison
- `zig-out/tmp/preact_baseline/discovery-buttons/discovery-buttons_report.json` - LLM analysis

#### Run all Preact tests with LLM analysis
```bash
python run_preact_baseline_compare.py --llm --width 1024 --height 768
```

#### Run with custom timeouts (for slow components)
```bash
python run_preact_baseline_compare.py --case discovery-cards \
  --react-wait-ms 1000 \
  --script-timeout-ms 15000 \
  --llm
```

#### Skip specific tests
```bash
python run_preact_baseline_compare.py --skip counter game-ui --llm
```

#### Single frame capture (faster)
```bash
python run_preact_baseline_compare.py --case discovery-inbox --frames 1
```

### Command-Line Options

```
--preact-dir PATH          Directory containing preact-* test directories
                          (default: zig-out/bin/data)
--out-dir PATH            Output directory for comparison results
                          (default: zig-out/tmp/preact_baseline)
--bin-dir PATH            Directory containing html_render_test binary
                          (default: zig-out/bin)

--width N                 Viewport width in pixels (default: 1024)
--height N                Viewport height in pixels (default: 768)
--frames N                Number of frames to capture (default: 5)
--frame-ms N              Milliseconds between frames (default: 100)
--update                  Call dong_view_update() between frames

--base-wait-ms N          Initial wait before Chrome screenshot (default: 50ms)
--react-wait-ms N         Extra wait for React/Preact hydration (default: 500ms)
--script-timeout-ms N     Dong script execution timeout (default: 10000ms)

--llm                     Enable LLM-powered visual diff analysis
--case NAME              Run single test by name (e.g. discovery-buttons)
--skip NAME [NAME ...]   Skip specific test names

--help                   Show all options
```

## Workflow: From Component to Production

### 1. Create Component (JSX)
```
preact/examples/discovery-buttons/
  ├── main.jsx           # Your Preact component
  ├── bundle.js          # Auto-generated by build
  └── index.html         # Renders bundle.js
```

### 2. Build Bundle
```bash
cd dong/
zig build preact    # Compiles main.jsx → bundle.js via esbuild
```

### 3. Generate Baselines & Run Comparison
```bash
python scripts/tools/run_preact_baseline_compare.py --case discovery-buttons --llm
```

### 4. Review Results
- View merged comparison image: `discovery-buttons_merged.png`
- Analyze LLM report: `discovery-buttons_report.json`
- If severity = **NONE**: ✓ ready for production
- If severity = **WARNING**: Review with `/preact-baseline-comparison` skill
- If severity = **CRITICAL**: Bug in Dong engine or bundle—investigate

### 5. Deploy
Once all tests pass (NONE or acceptable WARNING), add component to test suite for regression detection.

## Troubleshooting

### Issue: "No preact tests found"
**Cause**: Build didn't generate bundle.js files
**Fix**:
```bash
zig build preact  # Rebuild preact bundles
```

### Issue: Severity always "UNKNOWN"
**Cause**: Not using `--llm` flag
**Fix**:
```bash
python run_preact_baseline_compare.py --case discovery-buttons --llm
```

### Issue: Chrome baseline is blank/empty
**Cause**: HTML file not found or Playwright not installed
**Fix**:
```bash
# Verify file exists:
ls -la zig-out/bin/data/preact-discovery-buttons/index.html

# Install Playwright:
pip install playwright
playwright install chromium
```

### Issue: Dong render is entirely black
**Cause**: Bundle not loading, JS timeout, or hydration failure
**Fix**:
```bash
# Increase timeout and enable debug logging:
DONG_LOG_LEVEL=DEBUG DONG_SCRIPT_TIMEOUT_MS=15000 \
  python run_preact_baseline_compare.py --case discovery-buttons
```

### Issue: Encoding errors on Windows
**Cause**: Python using system locale (GBK) instead of UTF-8
**Fix**: Should be auto-fixed by latest version; if not:
```python
# Add to top of run_preact_baseline_compare.py
import io
import sys
if sys.platform == "win32":
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')
```

## Advanced Topics

### Custom Baseline Wait Time

If your component needs extra time for animations or async operations:

```bash
python run_preact_baseline_compare.py --case my-component \
  --react-wait-ms 2000 --base-wait-ms 100 --llm
```

This gives Chrome 2.1 seconds total before screenshot, and Dong frames start at 100ms intervals.

### Multi-viewport Testing

Capture same component at multiple sizes:

```bash
for size in "800x600" "1024x768" "1920x1080"; do
  w=${size%x*}
  h=${size#*x}
  python run_preact_baseline_compare.py --case discovery-buttons \
    --width $w --height $h --llm \
    --out-dir "zig-out/tmp/preact_baseline_${w}x${h}"
done
```

### Frame-by-frame Animation Analysis

Capture more frames to see animation progression:

```bash
python run_preact_baseline_compare.py --case discovery-buttons \
  --frames 20 --frame-ms 50 --llm
```

This captures 20 frames at 50ms intervals = 1 second of animation.

### Comparing Git Branches

```bash
# On main branch:
python run_preact_baseline_compare.py --case discovery-buttons --llm \
  --out-dir "/tmp/preact_baseline_main"

# Switch branch:
git checkout feature/my-rendering-fix
zig build preact

# On feature branch:
python run_preact_baseline_compare.py --case discovery-buttons --llm \
  --out-dir "/tmp/preact_baseline_feature"

# Compare reports:
diff /tmp/preact_baseline_main/discovery-buttons/discovery-buttons_report.json \
     /tmp/preact_baseline_feature/discovery-buttons/discovery-buttons_report.json
```
