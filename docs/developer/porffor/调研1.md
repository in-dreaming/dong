基于当前 Porffor 集成（`enjin_porffor` 分支 ≈ 上游 0.61.13）与 QuickJS 完整绑定层的对比，整理如下。

---

## 现状速览

**Porffor 路径已实现（P0）：**

| 能力 | 实现方式 |
|------|----------|
| AOT 模块启动 | manifest + `data-porffor-module` |
| `getElementById` | 返回 `nodeId`（非 DOM 对象） |
| `setTextContent` / 读文本 | stage/commit import |
| `addEventListener` | 仅 **export 名字符串**，handler 为独立子模块 |
| `setTimeout` | 同上，export 名回调 |
| `dong_time_now` / 日志 | host import |
| 定时器 tick | C++ `PorfforHost::processTimers` |

**QuickJS 路径仍有、Porffor 完全缺失：** 约 150 个带 `<script>` 的 HTML 测试所依赖的全套 Web API。

---

## 一、语言 / 运行时模型（Porffor 固有缺口）

这些是 [Porffor README](https://github.com/in-dreaming/porffor) 明确列出的限制，也是 Dong 与浏览器 JS 差距最大的部分。

| 缺失特性 | Dong 中的用途 | 替代方案 | 在 Porffor 中实现的可能性 |
|----------|---------------|----------|---------------------------|
| **`eval()` / `Function()` / 内联 `<script>`** | HTML 测试、React bundle、`html_render_test --eval-after-frame0` | ① 继续用 QuickJS 构建跑回归<br>② 脚本全部迁入 manifest<br>③ 交互改 C++ 驱动 + 多帧渲染 | **极低** — AOT 设计根约束，不适合改 |
| **跨作用域闭包** | 几乎所有事件 handler、`.then()`、React Hooks | ① 全局变量 + 具名 export handler<br>② 逻辑下沉 C++ | **极低** — README: *No variables between scopes (except args and globals)* |
| **函数引用作回调** | `addEventListener(el, 'click', fn)` | export 名字符串 + 子模块（已实现） | **不适用** — 应走 Dong host 模型，而非改 Porffor |
| **ES Module / `import`** | React/Preact bundle、`<script type="module">` | manifest 多模块 + registry | **低** — 无运行时 loader；build 期 bundling 可部分替代 |
| **`Promise` / `async-await`** | `fetch().then()`、现代测试 | ① QuickJS<br>② 同步 host import + 轮询<br>③ 状态机式 AOT 脚本 | **低~中** — 上游有实现但 *known bugs*；游戏 UI 更宜 callback import |
| **完整 JS 对象模型** | `element.value`、`element.style.color` | 扁平 prelude 函数 + nodeId | **中（Dong 侧）** — 不必在 Porffor 做 Proxy；扩 host import 即可 |
| **对象方法调用不稳定** | `el.addEventListener(...)` | 扁平函数（prelude 已采用） | **中（Dong 侧 workaround）** — 2c 路径问题，stage/commit 可扩展 |

## 二、Dong 集成层缺口（C++ / prelude 可补）

| 缺失特性 | QuickJS 有 | 替代方案 | 在 Porffor 中实现的可能性 |
|----------|-----------|----------|---------------------------|
| **2c 多参数 import** | N/A | `dong_stage_0/1/2` + `commit_*`（已用） | **高（Dong 侧）** — 继续扩 stage 槽位或拆成单参 import |
| **内联 `onclick=` 等** | `scanAndRegisterInlineEventHandlers` | build 期从 HTML 生成 handler 模块 | **高（工具链）** — 不需改 Porffor |
| **运行时 `dong_engine_eval_script`** | ✓ | 预编译 snippet 模块 | **不适用** — 故意禁用 |
| **多 View `dong.getView()`** | ✓ | 每 View 独立 manifest 模块 | **高（Dong 侧）** |
| **Event 对象传给 handler** | MouseEvent/KeyboardEvent 等 | handler 前写入全局 event 槽（stage 模式） | **高（Dong 侧）** |
| **`requestAnimationFrame`** | ✓ tick 队列 | C++ 每帧 callExport；或 host import 注册 export | **高（Dong 侧）** |
| **`setInterval`** | ✓ | 同 setTimeout 扩展 | **高（Dong 侧）** |
| **fetch 完成回调** | Promise 链 | `dong_fetch_start` + 下一帧 `dong_fetch_poll` 或完成时 callExport | **高（Dong 侧）** — 避开 Promise |

## 三、DOM / Web API 缺口（按优先级）

### P1 — 基础交互（表单、简单页面）

| API 类别 | 代表能力 | 测试示例 | 替代方案 | Porffor/Host 可行性 |
|----------|----------|----------|----------|---------------------|
| **元素属性** | `.value`、`.checked`、`.disabled`、`.innerHTML` | `test_simple_bindings.html` | `getValue(nodeId)` / `setValue(nodeId, v)` import | **高** — host import |
| **查询** | `querySelector`、`getElementsByTagName` | 大量测试 | 返回 nodeId 或 id 字符串 | **高** |
| **DOM 修改** | `appendChild`、`remove`、`createElement` | `test_dom_manipulation.html` | 扁平 import + nodeId | **中~高** |
| **表单** | `submit()`、`checkValidity()`、`form.elements` | 表单测试集 | 拆成多个 import；复杂集合用 JSON 字符串返回 | **中** |
| **样式读写** | `element.style.setProperty`、`getComputedStyle` | CSSOM 测试 | import 传 property 名字符串 | **中** |
| **classList** | add/remove/toggle | 多处 | 四个 import 即可 | **高** |
| **事件类型扩展** | `input`、`change`、`keydown` + event 字段 | `test_select_keyboard.html` | 已有 bridge；补 event stage | **高（Dong 侧）** |
| **console.log** | 结构化日志 | 几乎所有 JS 测试 | `dongLog`（已有） | **高** |
| **performance.now** | 基准测试 | `test_performance_now.html` | `dong_time_now`（已有） | **高** |

### P2 — 异步 / 平台 API

| API | 用途 | 替代方案 | 可行性 |
|-----|------|----------|--------|
| **`fetch()`** | 本地/远程资源 | callback import 或 poll；QuickJS 回归 | **中（Host）** / **低（原生 Promise 脚本）** |
| **`navigator.clipboard`** | 剪贴板测试 | `dong_clipboard_read/write` import | **高（Host）** |
| **`matchMedia` / `CSS.supports`** | 响应式/CSS 测试 | 同步 import 返回 bool | **高（Host）** |
| **Selection / Range** | 选区、CE 测试 | 复杂；短期 QuickJS | **低~中** |
| **MutationObserver** | DOM 变更监听 | C++ 侧直接 callExport | **中（Host 驱动）** |
| **DOMParser** | 解析 HTML 片段 | C++ import 返回 root nodeId | **中** |
| **FormData** | 表单序列化 | 拆 import 或 JSON | **中** |
| **Video 事件/属性** | 媒体测试 | 已有 `dispatchMediaEvent` 骨架 | **中（Host）** |
| **Dialog** | showModal/close | import 包装 | **高（Host）** |

### P3 — 游戏 / Dong 专有

| API | 用途 | 替代方案 | 可行性 |
|-----|------|----------|--------|
| **`dong.textLayout` / overlay 绘制** | 游戏 UI | 专用 import 集 | **高（Host）** — 与 DOM 无关 |
| **Scene graph API** | `test_scene_graph.html` | import 或独立 C++ 模块 | **中~高（Host）** |
| **IME / composition / beforeinput** | 中文输入、CE | 事件 stage + export；逻辑多在 C++ | **中** — Porffor handler 只做 UI 反馈 |

## 四、框架与测试基础设施

| 缺失 | 影响 | 替代方案 | 可行性 |
|------|------|----------|--------|
| **React / Preact** | 全部示例 | QuickJS 构建；或预编译为极特殊 AOT（不现实） | **极低** |
| **闭包 + 虚拟 DOM** | 框架核心 | 不用 Porffor 跑框架 | **极低** |
| **150 个 HTML `<script>` 测试** | CI 回归 | 129 个无 script 测试仍可做静态渲染；JS 测试走 QuickJS 或逐条迁移 | **迁移：<10%** 可迁 |
| **headless eval snippet** | CE bold 等多帧测试 | 每 snippet 一个 manifest handler；或 C++ 模拟输入 | **中（工具链）** |

## 五、2c / 工具链层（非语义但阻塞开发效率）

| 问题 | 影响 | 替代 | 在 Porffor fork 修复可能性 |
|------|------|------|---------------------------|
| **多参数 wasm import 代码生成** | 必须 stage/commit | 继续 Dong workaround | **高** — `enjin_porffor` 可专向修 2c |
| **NaN / Infinity 字面量** | 编译 C 失败 | 后处理替换（已做） | **高** — 可上游或 fork 修 |
| **`log` 与 `math.h` 冲突** | 符号冲突 | 改名 `dongLog`（已做） | **高** |
| **每模块 helper 需 `static`** | 链接 duplicate symbol | 后处理（已做） | **中** — 2c 可生成 static |
| **编译 stdout 噪声** | 开发体验 | compile API 加 `--quiet` 或 Dong 侧截断 | **高** |

## 六、建议路线图（按投入产出）

```
                    可行性
Host import 扩展  ████████████  高 — 主战场
Build 工具链      ██████████░░  高 — HTML→manifest、porffor-skip
2c fork 修复      ████████░░░░  中~高 — 减少 stage 样板
Promise/async 脚本 ███░░░░░░░░  低 — 避开，用 callback
闭包 / eval / ES模块 ██░░░░░░░░░  极低 — 不改 Porffor，双引擎并存
React/Preact      █░░░░░░░░░░░  极低 — 永久 QuickJS
```

**短期（可支撑简单游戏 UI）：**  
扩展 prelude：`getValue`/`setValue`、`querySelector`、`classList_*`、更多事件类型的 event stage、`setInterval`/`rAF`、fetch callback 版。

**中期（覆盖更多测试）：**  
HTML 内联事件 build 期提取；`porffor-skip` + CI 双引擎；多帧测试改为「预编译 handler + 宿主发事件」。

**长期（若要坚持单引擎）：**  
在 [in-dreaming/porffor](https://github.com/in-dreaming/porffor) fork 修 2c import、改善同文件多 `main`/export；**不要**指望 upstream 支持闭包/eval。

**务实结论：**  
Porffor 适合 **无闭包、build 期确定的脚本**（状态机式 UI、benchmark、简单 DOM）。完整 Web 语义应继续由 **QuickJS 路径** 承担，或通过 **Host import 扁平 API** 在 Porffor 上重建子集，而不是等 Porffor 变成完整 JS 引擎。
