# T18 — 框架与组件库：调研与规格定稿（替代 React / Preact）

- **Area**: toolchain
- **性质**: **调研/规格型** — 产出《框架规格书》，验收 = 评审通过；**不合入生产代码**（spike 验证代码不合入）。实现由 T22（编译器/运行时）、T23（组件库）按本规格执行
- **优先级**: P1（战略级：React/Preact 永久退役的前提）
- **依赖**: 无——可立即启动。规格中涉及的状态模型 / 字符串协议按 T15/T16 的定稿结论引用；若其未定稿，规格中留接口占位并标注
- **预估**: 3–5 人天（含评审一轮）
- **前置阅读**: `docs/developer/porffor/tasks/setup.md` §1 §4 §7.0；`调研1.md`（框架不可迁移的原因）

## 背景

全面切换后 React / Preact 永久退役（依赖闭包 + 运行时动态性，Porffor 不可能支持），但「组件化 + 声明式 + 单向数据流」的开发体验必须保留。本任务**只做调研与规格**，把后续实现变成「照规格施工」——避免实现任务边写边发明语义。

## 复刻目标（规格的两个锚点，不允许自由发挥）

1. **语法对标：Svelte 语法的明确子集**。模板语法**不自创**，从 Svelte 现有语法中圈定一个子集（插值 `{expr}`、`on:event`、`{#if}` / `{#each}`、组件标签与 props 等），逐条列入规格：**支持 / 不支持 / 与 Svelte 的语义差异**（差异主要来自 Porffor 限制：无闭包 → 事件值只能是具名 export；无运行时组件实例 → 编译期内联展开）。好处：语法有现成文档与社区认知，行为争议以 Svelte 文档 + REPL 输出为参照。
2. **行为对标：现有三个 React 示例的 1:1 复刻**。`examples/data/react-counter`、`react-todo-classic`、`react-game-ui` 是验收基准：规格书为每个示例列出逐屏功能清单（交互、状态、渲染效果），T22/T23 的验收就是按清单复刻 + 多帧截图比对。

## 规格书必须包含的章节（《framework-spec.md》）

1. **语法子集表**：每条构造一行——语法、语义、Svelte 文档章节引用、限制与差异、**「输入模板 → 生成 Porffor JS」黄金样例**（这些样例直接成为 T22 编译器快照测试的初始集）。
2. **store 与更新机制**：状态存放形态（模块全局，依赖 T15 方案 A；或 host 状态槽，T15 方案 B——两种形态的生成代码差异写清）；`setState` → 脏标记 → 编译期生成的「key → 更新语句」映射；更新粒度（第一版粗粒度：一个 key 触发一组更新语句）。
3. **事件模型**：`on:click={inc}` → `addEventListener(nodeId, 'click', 'inc')`（T08 export 名模型）；事件字段读取走 T08 事件槽；handler 命名与冲突规则。
4. **列表渲染语义**：`{#each}` 的 key 规则、第一版支持范围（尾部增删 + 按 key 更新）、超出范围的行为（重建）。
5. **组件组合**：编译期内联展开规则、props 传递（编译期替换 + store 子树）、不支持的动态场景清单。
6. **编译器与工具链集成**：CLI 形态、在 `porffor_compile.mjs` / manifest 管线中的位置、产物布局、与 T12 内联 handler 生成器的代码复用边界。
7. **组件库清单**（T23 的施工图）：覆盖三个对标示例所需的全部组件（button / label / list / progress / dialog / input 等），每个组件给出 props 表、DOM 结构契约、样式契约、在对标示例中的用法举例。
8. **禁止面与 lint 规则**：生成代码不得含闭包捕获 / eval / Promise / 动态 import 的机器可查规则。
9. **三个示例的复刻 checklist**：逐屏功能点（T22/T23 验收即勾选此表）。

## 验收标准

1. 《framework-spec.md》章节完整（上述 9 节），评审通过并记录评审意见与决议。
2. 语法子集表每条构造都有黄金样例；样例经 spike 验证「生成的 JS 能被 `porf c` 编译」（spike 代码不合入，验证记录合入）。
3. 三个示例的复刻 checklist 完整（作为 T22/T23 的验收清单）。
4. 与 T15/T16 的依赖点显式标注（哪些章节等其定稿、当前按什么假设写）。
